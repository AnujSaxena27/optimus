Setup\n\n1. **Clone and Install**\n   ```bash\n   git clone <repo-url>\n   cd sepmpro\n   npm install\n   ```\n\n2. **Configure Environment**\n   ```bash\n   cp .env.example .env.local\n   # Edit .env.local with your credentials\n   ```\n\n3. **Install nanoid**\n   ```bash\n   npm install nanoid\n   ```\n\n4. **Run Development Server**\n   ```bash\n   npm run dev\n   ```\n   Visit: http://localhost:3000\n\n---\n\n## Docker Deployment\n\n### Build Docker Image\n\n```bash\n# Development build\ndocker build -t inference-optimizer:latest .\n\n# Build with specific tag\ndocker build -t inference-optimizer:v1.0.0 .\n\n# Build for production\ndocker build --target production -t inference-optimizer:prod .\n```\n\n### Run Docker Container\n\n```bash\n# Basic run\ndocker run -p 3000:3000 \\\n  -e GROQ_API_KEY=your_key \\\n  -e NEXT_PUBLIC_SUPABASE_URL=your_url \\\n  -e NEXT_PUBLIC_SUPABASE_ANON_KEY=your_key \\\n  inference-optimizer:latest\n\n# With volume mounting for logs\ndocker run -p 3000:3000 \\\n  -v $(pwd)/logs:/app/logs \\\n  -e GROQ_API_KEY=your_key \\\n  inference-optimizer:latest\n\n# With custom configuration\ndocker run -p 3000:3000 \\\n  -e NODE_ENV=production \\\n  -e LOG_LEVEL=INFO \\\n  -e BATCH_SIZE=10 \\\n  -e GROQ_API_KEY=your_key \\\n  inference-optimizer:latest\n```\n\n### Docker Compose\n\n```bash\n# Start all services\ndocker-compose up -d\n\n# View logs\ndocker-compose logs -f app\n\n# Stop services\ndocker-compose down\n\n# Rebuild images\ndocker-compose build --no-cache\n```\n\n#### With Environment File\n\n```bash\n# Create .env file\ncat > .env << Security Best Practices\n\n### 1. Environment Variables\n\n✅ **DO**:\n- Use `.env.local` for development\n- Use secure secret management in production\n- Rotate API keys regularly\n- Use different keys for dev/prod\n\n❌ **DON'T**:\n- Commit `.env.local` to git\n- Use same key across environments\n- Share API keys in logs\n- Hardcode secrets in code\n\n### 2. JWT Security\n\n✅ **DO**:\n- Verify JWT expiration\n- Validate JWT signature with public key\n- Use HTTPS for all requests\n- Set appropriate JWT expiry\n\n❌ **DON'T**:\n- Trust JWT without verification\n- Send sensitive data in JWT\n- Use expired tokens\n\n### 3. Rate Limiting\n\n✅ **DO**:\n- Implement per-user rate limits\n- Use sliding window algorithm\n- Return 429 when limited\n- Log rate limit violations\n\n❌ **DON'T**:\n- Allow unlimited requests\n- Use small rate limit windows\n- Block IP addresses (use user IDs)\n\n### 4. API Security\n\n✅ **DO**:\n- Validate all input\n- Sanitize error messages\n- Use HTTPS/TLS\n- Set security headers\n\n❌ **DON'T**:\n- Expose stack traces\n- Log sensitive data\n- Allow CORS from everywhere\n- Trust client-provided user IDs\n\n### 5. Docker Security\n\n✅ **DO**:\n- Use non-root user\n- Scan images for vulnerabilities\n- Keep base images updated\n- Use specific version tags\n\n```dockerfile\nRUN addgroup -g 1001 -S nodejs && adduser -S nextjs -u 1001\nUSER nextjs\n```\n\n---\n\n## Monitoring & Maintenance\n\n### Logs Analysis\n\n```bash\n# View recent errors\ndocker logs --tail 100 -f <container-id> | grep ERROR\n\n# Export logs\ndocker logs <container-id> > app.log 2>&1\n```\n\n### Health Monitoring\n\n```bash\n# Check container health\ndocker ps --format \"{{.Names}}\\t{{.Status}}\"\n\n# Test API endpoint\ncurl -H \"Authorization: Bearer $(cat jwt_token.txt)\" \\\n     http://localhost:3000/api/models\n```\n\n### Performance Monitoring\n\n```bash\n# Monitor resource usage\ndocker stats\n\n# Check container processes\ndocker top <container-id>\n```\n\n### Database Maintenance\n\n```sql\n-- Clean up old logs (keep 30 days)\nDELETE FROM request_logs \nWHERE created_at < NOW() - INTERVAL '30 days';\n\n-- Create indexes for performance\nCREATE INDEX idx_user_model ON request_logs(user_id, model);\nCREATE INDEX idx_created_range ON request_logs(created_at DESC);\n```\n\n### Update Strategy\n\n1. **Test Updates Locally**\n   ```bash\n   npm install\n   npm run build\n   npm run dev\n   ```\n\n2. **Push to Staging**\n   ```bash\n   git push origin staging\n   # Deploy to staging environment\n   ```\n\n3. **Test in Staging**\n   - Run integration tests\n   - Load test\n   - Verify all endpoints\n\n4. **Deploy to Production**\n   ```bash\n   git push origin main\n   # Use blue-green deployment for zero downtime\n   ```\n\n### Backup Strategy\n\n```bash\n# Backup Supabase data\nsupabase db pull --local\n\n# Store in version control\ngit add supabase/migrations\ngit commit -m \"Backup: $(date)\"\ngit push\n```\n\n---\n\n## Troubleshooting\n\n### Container Won't Start\n\n```bash\n# Check logs\ndocker logs <container-id>\n\n# Verify environment variables\ndocker inspect <container-id> | grep -A 20 '\"Env\"'\n\n# Check resource constraints\ndocker stats --no-stream\n```\n\n### High Memory Usage\n\n```bash\n# Reduce cache size\n-e CACHE_MAX_ENTRIES=5000\n\n# Reduce queue size\n-e MAX_QUEUE_SIZE=500\n\n# Monitor memory\ndocker stats --no-stream --format \"{{.Names}}\\t{{.MemUsage}}\"\n```\n\n### API Timeouts\n\n1. Check Groq API status\n2. Increase `REQUEST_TIMEOUT_MS`\n3. Check network connectivity\n4. Review error logs\n\n### Database Connection Issues\n\n1. Verify Supabase credentials\n2. Check network access\n3. Test connection manually:\n   ```bash\n   npx supabase link\n   ```\n\n---\n\nFor more information, see [ARCHITECTURE.md](./ARCHITECTURE.md) and [CONFIGURATION.md](./CONFIGURATION.md)\n
