# API Documentation\n\n## Overview\n\nThe Inference Optimizer API provides optimized inference endpoints with caching, batching, and request tracking functionality.\n\n## Base URL\n\n```\nhttps://api.inference.example.com\n```\n\n## Authentication\n\nAll endpoints (except `/health`) require a valid Supabase JWT token in the `Authorization` header:\n\n```\nAuthorization: Bearer <JWT_TOKEN>\n```\n\n## Response Format\n\n### Success Response\n\n```json\n{\n  \"success\": true,\n  \"status\": 200,\n  \"data\": { /* endpoint-specific data */ },\n  \"timestamp\": \"2024-01-15T10:30:45.123Z\"\n}\n```\n\n### Error Response\n\n```json\n{\n  \"success\": false,\n  \"status\": 400,\n  \"error\": \"Error message\",\n  \"code\": \"ERROR_CODE\",\n  \"timestamp\": \"2024-01-15T10:30:45.123Z\"\n}\n```\n\n---\n\n## Endpoints\n\n### 1. Chat Completion\n\nGenerate text using the selected model with automatic batching and caching.\n\n**Endpoint**: `POST /api/chat`\n\n**Authentication**: Required (JWT)\n\n**Request Body**:\n\n```json\n{\n  \"message\": \"What is machine learning?\",\n  \"model\": \"mixtral-8x7b-32768\",\n  \"temperature\": 0.7,\n  \"maxTokens\": 1024\n}\n```\n\n**Parameters**:\n\n| Parameter | Type | Default | Description |\n|-----------|------|---------|-------------|\n| `message` | string | *required* | The input prompt (max 10,000 characters) |\n| `model` | string | *required* | Model ID (e.g., 'mixtral-8x7b-32768') |\n| `temperature` | number | 0.7 | Sampling temperature (0.0 - 2.0) |\n| `maxTokens` | number | 1024 | Maximum tokens to generate (1 - model's max) |\n\n**Response**:\n\n```json\n{\n  \"success\": true,\n  \"status\": 200,\n  \"data\": {\n    \"id\": \"req_abc123def456\",\n    \"userId\": \"user_123\",\n    \"model\": \"mixtral-8x7b-32768\",\n    \"reply\": \"Machine learning is a subset of artificial intelligence...\",\n    \"tokensUsed\": 156,\n    \"processingTimeMs\": 145,\n    \"cached\": false,\n    \"timestamp\": \"2024-01-15T10:30:45.123Z\"\n  },\n  \"timestamp\": \"2024-01-15T10:30:45.123Z\"\n}\n```\n\n**Status Codes**:\n\n| Code | Meaning |\n|------|----------|\n| 200 | Success |\n| 400 | Bad request (invalid parameters) |\n| 401 | Unauthorized (missing/invalid JWT) |\n| 429 | Rate limit exceeded |\n| 500 | Server error |\n\n**Examples**:\n\n```bash\n# Basic request\ncurl -X POST http://localhost:3000/api/chat \\\n  -H \"Content-Type: application/json\" \\\n  -H \"Authorization: Bearer $JWT_TOKEN\" \\\n  -d '{\n    \"message\": \"Hello!\",\n    \"model\": \"mixtral-8x7b-32768\"\n  }'\n\n# With custom parameters\ncurl -X POST http://localhost:3000/api/chat \\\n  -H \"Content-Type: application/json\" \\\n  -H \"Authorization: Bearer $JWT_TOKEN\" \\\n  -d '{\n    \"message\": \"Write a poem about AI\",\n    \"model\": \"llama-3-70b-8192\",\n    \"temperature\": 0.9,\n    \"maxTokens\": 512\n  }'\n```\n\n**Error Examples**:\n\nMissing required field:\n```json\n{\n  \"success\": false,\n  \"status\": 400,\n  \"error\": \"Message is required\"\n}\n```\n\nRate limit exceeded:\n```json\n{\n  \"success\": false,\n  \"status\": 429,\n  \"error\": \"Rate limit exceeded\",\n  \"retryAfter\": 45\n}\n```\n\n---\n\n### 2. List Models\n\nFetch available models with metrics and pricing information.\n\n**Endpoint**: `GET /api/models`\n\n**Authentication**: Optional (can be public)\n\n**Query Parameters**: None\n\n**Response**:\n\n```json\n{\n  \"success\": true,\n  \"status\": 200,\n  \"data\": [\n    {\n      \"id\": \"mixtral-8x7b-32768\",\n      \"name\": \"mixtral-8x7b-32768\",\n      \"object\": \"model\",\n      \"created\": 1700000000,\n      \"owned_by\": \"mistral-community\",\n      \"maxTokens\": 32768,\n      \"costPerMTok\": 0.24,\n      \"latencyMs\": 150,\n      \"provider\": \"Groq\"\n    },\n    {\n      \"id\": \"llama-3-70b-8192\",\n      \"name\": \"llama-3-70b-8192\",\n      \"object\": \"model\",\n      \"created\": 1700000000,\n      \"owned_by\": \"meta-llama\",\n      \"maxTokens\": 8192,\n      \"costPerMTok\": 0.59,\n      \"latencyMs\": 200,\n      \"provider\": \"Groq\"\n    }\n  ],\n  \"timestamp\": \"2024-01-15T10:30:45.123Z\"\n}\n```\n\n**Notes**:\n- Results are cached for 1 hour\n- Includes enriched metadata (cost, latency, etc.)\n- Fallback data returned if API unavailable\n\n**Examples**:\n\n```bash\n# Get available models\ncurl http://localhost:3000/api/models\n\n# Parse model IDs\ncurl http://localhost:3000/api/models | jq '.data[].id'\n```\n\n---\n\n### 3. Health Check\n\nCheck if the API is running and healthy.\n\n**Endpoint**: `GET /health` (or any endpoint)\n\n**Authentication**: Not required\n\n**Response**: `200 OK` if running\n\n**Examples**:\n\n```bash\n# Check health\ncurl http://localhost:3000/health\n\n# In Docker health check\nwget --quiet --tries=1 --spider http://localhost:3000\n```\n\n---\n\n## Request Lifecycle\n\n1. **JWT Verification**\n   - Token extracted from Authorization header\n   - Token expiration checked\n   - User ID extracted from JWT payload\n\n2. **Rate Limiting**\n   - User's request count verified\n   - If under limit: continue, increment counter\n   - If over limit: return 429\n\n3. **Input Validation**\n   - Message length checked (max 10,000 chars)\n   - Model ID validated\n   - Parameters validated\n\n4. **Cache Lookup**\n   - Check if identical request cached\n   - If hit: return cached response immediately\n   - If miss: continue to queue\n\n5. **Request Queuing**\n   - Add to request queue\n   - Wait for batch accumulation or timeout\n   - Process with other requests in batch\n\n6. **Groq API Call**\n   - Send batch to Groq API\n   - Wait for response\n   - Parse and validate response\n\n7. **Response Caching**\n   - Store response in cache\n   - Set expiration (TTL)\n\n8. **Logging**\n   - Write request log to database\n   - Track user activity\n   - Calculate costs\n\n9. **Return Response**\n   - Format standardized response\n   - Include metadata (request ID, processing time, etc.)\n   - Return to client\n\n---\n\n## Rate Limiting\n\n### Limits\n\n- **Default**: 100 requests per minute per user\n- **Sliding Window**: 1-minute rolling window\n- **Status Code**: 429 Too Many Requests\n\n### Response Header\n\n```\nRateLimitRemaining: 45\n```\n\n### Example\n\n```bash\n# Request 101st time in a minute\ncurl -X POST http://localhost:3000/api/chat \\\n  -H \"Authorization: Bearer $JWT_TOKEN\" \\\n  -d '{...}'\n\n# Response:\n# HTTP/1.1 429 Too Many Requests\n# {\n#   \"success\": false,\n#   \"status\": 429,\n#   \"error\": \"Rate limit exceeded\",\n#   \"retryAfter\": 45\n# }\n```\n\n---\n\n## Caching\n\n### How It Works\n\n- Responses cached based on: `model + message + temperature`\n- Default TTL: 1 hour\n- Automatic LRU eviction when full\n- Updates when new identical request comes\n\n### Cache Hit Response\n\n```json\n{\n  \"data\": {\n    \"cached\": true,\n    \"processingTimeMs\": 2\n  }\n}\n```\n\n### Override Cache (Future)\n\n```bash\ncurl -X POST http://localhost:3000/api/chat \\\n  -H \"Cache-Control: no-cache\" \\\n  -d '{...}'\n```\n\n---\n\n## Model Selection Guide\n\n### Fast & Cheap\n- `llama-3-8b-8192`: Fastest, lowest cost\n- Use for: Simple queries, low latency requirements\n\n### Balanced\n- `mixtral-8x7b-32768`: Good balance\n- Use for: Most applications\n\n### Accurate & Capable\n- `llama-3-70b-8192`: Most powerful\n- Use for: Complex reasoning, quality matters more\n\n### Cost Table\n\n| Model | Cost/MTok | Latency | Context |\n|-------|-----------|---------|----------|\n| llama-3-8b-8192 | $0.05 | 150ms | 8K |\n| mixtral-8x7b-32768 | $0.24 | 150ms | 32K |\n| llama-3-70b-8192 | $0.59 | 200ms | 8K |\n\n---\n\n## Error Handling\n\n### Common Errors\n\n#### 400 - Bad Request\n```json\n{\n  \"error\": \"Message exceeds maximum length of 10000 characters\"\n}\n```\n\n#### 401 - Unauthorized\n```json\n{\n  \"error\": \"Invalid or expired JWT token\"\n}\n```\n\n#### 429 - Rate Limited\n```json\n{\n  \"error\": \"Rate limit exceeded\",\n  \"retryAfter\": 45\n}\n```\n\n#### 500 - Server Error\n```json\n{\n  \"error\": \"Internal server error\",\n  \"requestId\": \"req_abc123\"\n}\n```\n\n### Retry Strategy\n\n1. **4xx Errors**: Don't retry (client error)\n2. **429 Errors**: Wait `retryAfter` seconds, then retry\n3. **5xx Errors**: Exponential backoff (1s, 2s, 4s, 8s)\n4. **Timeout**: Retry after REQUEST_TIMEOUT_MS\n\n---\n\n## Performance Metrics\n\n### Typical Response Times\n\n- **Cache Hit**: 2-5ms\n- **First Request**: 150-300ms\n- **Batch Process**: 200-500ms\n\n### Throughput\n\n- **Without Batching**: 10-20 requests/second\n- **With Batching**: 100-300 requests/second\n- **With Caching**: 1000+ requests/second (hit rate ~70%)\n\n---\n\n## Example Workflows\n\n### Simple Chat Request\n\n```bash\n#!/bin/bash\n\nJWT_TOKEN=\"your_jwt_token_here\"\nAPI_URL=\"http://localhost:3000\"\n\n# Request\nRESPONSE=$(curl -s -X POST \"$API_URL/api/chat\" \\\n  -H \"Content-Type: application/json\" \\\n  -H \"Authorization: Bearer $JWT_TOKEN\" \\\n  -d '{\n    \"message\": \"What is AI?\",\n    \"model\": \"mixtral-8x7b-32768\"\n  }')\n\n# Parse response\necho \"$RESPONSE\" | jq '.data.reply'\necho \"Processing time: $(echo \"$RESPONSE\" | jq '.data.processingTimeMs')ms\"\necho \"Cached: $(echo \"$RESPONSE\" | jq '.data.cached')\"\n```\n\n### Batch Processing\n\n```python\nimport requests\nimport concurrent.futures\n\nJWT_TOKEN = \"your_jwt_token_here\"\nAPI_URL = \"http://localhost:3000\"\n\ndef send_request(question):\n    response = requests.post(\n        f\"{API_URL}/api/chat\",\n        json={\n            \"message\": question,\n            \"model\": \"mixtral-8x7b-32768\"\n        },\n        headers={\"Authorization\": f\"Bearer {JWT_TOKEN}\"}\n    )\n    return response.json()[\"data\"]\n\nquestions = [\n    \"What is machine learning?\",\n    \"How does AI work?\",\n    \"What is deep learning?\"\n]\n\nwith concurrent.futures.ThreadPoolExecutor(max_workers=3) as executor:\n    results = list(executor.map(send_request, questions))\n\nfor result in results:\n    print(f\"Response: {result['reply'][:100]}...\")\n    print(f\"Processing time: {result['processingTimeMs']}ms\\n\")\n```\n\n---\n\nFor more information, see [ARCHITECTURE.md](./ARCHITECTURE.md) and
